UX / perf:
- U1: SessionDetailsPanel.vue used to re-fire all 5 RPCs on every
  chat-tab switch. Split into global loaders (builtinTools + quota,
  loaded once on mount) and per-session loaders (skills + usage +
  mcp + plan, still re-fetch). Cuts RPC traffic on tab switch by 40%.
- U2: warnedThresholds Set no longer reset on session switch.
  Quota is account-wide; a 90% warning that already fired shouldn't
  re-fire because the user clicked a different tab.
- U3: openSessionsByDefault now console.warn's on retry exhaustion
  instead of silently bailing. Surfaces in the in-app log viewer.
- U6: cwdFor re-checks the entry's workingDirectory after each
  await before write. Avoids a concurrent-writer stale overwrite.
- U7: removePending resolves target requestId from the ambient queue
  first, then removes by id from both lists. Was scanning both
  independently by kind and could remove different entries when
  multiple pending requests of the same kind existed.
- U8: messageHandlers user-message dedup uses a manual backwards
  for loop with early break instead of [...items].reverse().find().
  Avoids the array copy on every user.message (and the entire
  history replay on resume).
