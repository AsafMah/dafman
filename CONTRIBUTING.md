## Development setup
```bash
git clone https://github.com/<owner>/dafman.git
cd dafman
npm install
npm run tauri dev
```
Run the test suites and quality gates via npm scripts (the same ones CI uses):
```bash
npm run check       # lint (vue-tsc + clippy + fmt) + all tests + build
npm test            # frontend unit tests (vitest)
npm run test:rust   # backend tests (cargo test)
npm run test:all    # both
npm run lint        # vue-tsc + cargo fmt --check + cargo clippy -D warnings
npm run fmt:rust    # auto-format Rust
```
## Branching & commits
- Branch from `main`.
- Use short, descriptive branch names: `feat/multi-account-auth`, `fix/dark-mode-bubbles`, `docs/roadmap-update`.
- Conventional commits encouraged but not required. Examples:
  - `feat: per-session GitHub auth pinning`
  - `fix(chat): handle session.idle before final message`
  - `docs(plans): integrate Supercharged v2.0 features`
- Keep commits focused. Squash before merge unless history is meaningful.

## Code style
- **Rust:** `cargo fmt`, `cargo clippy -- -D warnings`. No `unwrap` in command paths.
- **TypeScript:** strict mode. `vue-tsc --noEmit` must pass.
- **Vue:** components are dumb; data + actions live in composables / Pinia stores.
- **CSS:** prefer PrimeVue tokens (`var(--p-*)`); avoid hardcoded hex except for per-session accents.
## Architecture conventions
- Backend domains expose traits + impls (see [`plans/plan-architecture.prompt.md`](plans/plan-architecture.prompt.md)). Only `ipc/` and `app/` know about `tauri::*`.
- Frontend stores call IPC; components don''t.
- Every Tauri command returns `Result<T, AppError>`.
- Every event payload has a typed mirror on the frontend (eventually generated by `tauri-specta`).
