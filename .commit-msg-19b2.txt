feat(library): Agents tab with create/delete CRUD (Phase 19b.2)

Third tab in the Library panel for managing filesystem-backed
custom agents (the .agent.md files the SDK auto-discovers from
~/.copilot/agents/ and <workspace>/.github/agents/).

What works:
- Lists existing agent files grouped by scope (Project / User).
- "+ New agent" button opens an inline form: scope radio, name,
  displayName (optional), description (required), tools (csv),
  skills (csv), model (optional), user-invocable toggle, prompt
  textarea.
- Submit writes <scope-root>/<name>.agent.md with YAML frontmatter
  and the prompt body, then calls session.rpc.agent.reload so the
  picker sees it immediately.
- Reveal button opens the file in the OS file manager.
- Delete button removes the file (with confirm).

Architecture:
- New src-bun/app/agentFiles.ts module:
  - Strict name validation: [A-Za-z0-9][A-Za-z0-9._-]{0,63}, rejects
    Windows reserved names (CON/PRN/AUX/NUL/COMx/LPTx), max 64 chars.
  - Scope path resolution with defense-in-depth root check.
  - Minimal YAML serializer (supports the simpler frontmatter keys;
    nested mcp-servers and github keys deferred to direct edits).
  - Atomic write (tmp file + rename).
  - Refuse-overwrite on writeAgent (Edit deferred).
- 4 new bun RPCs on SessionRegistry: listAgentFiles (Project +
  User), listAgentFilesGlobal (User only, for the no-session
  fallback), writeAgentFile, deleteAgentFile.
- Project scope writes use the session's cached workingDirectory
  (renderer can't supply an arbitrary path).

Create + delete only (no Edit) per rubber-duck blocking finding:
- SDK accepts unknown frontmatter keys our minimal writer can't
  serialize (mcp-servers nested objects, github toolsets).
- A write-only Edit would silently strip those.
- v1: users open the file directly via Reveal for advanced edits.

Rubber-duck adoptions (all 7 findings):
1. Server-side eligibility check via path resolution + root verify.
2. Create/delete only.
3. Strict name validation against traversal + Windows reserved.
4. session.background_tasks_changed in addition to subagent.* —
   already done in 19b.1.
5. Separate file-management RPCs from SDK agent.list (this commit).
6. New agentFiles module instead of growing SessionRegistry (this
   commit; SessionRegistry just delegates with sessionId-derived
   workingDirectory).
7. Split into 19b.1 + 19b.2 — done.

Tests (16 new):
- validateAgentName: happy paths + path traversal + Windows
  reserved names + empty/whitespace/leading-dot + oversize.
- writeAgent: frontmatter content, displayName omitted-when-equal,
  empty description rejected, refuse-overwrite, scope-without-wd
  rejected, traversal name throws before I/O.
- listAgentFiles: discovers .agent.md + .md, skips non-md, returns
  empty when dir missing.
- deleteAgent: removes file, returns false for missing, validates
  name before resolve.

Plus a small fix to e2e/full/flows/18-library-mcp.pwtest.ts: PrimeVue
Tabs renders every panel in the DOM (just hides inactive), so the
new Agents tab's hint text mentions of `.github/agents/` collided
with the MCP test's unscoped `text=github` locator. Scoped to the
active MCP panel.
